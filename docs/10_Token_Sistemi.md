\## 1. Sistem Amacı



Token sistemi, Yelkenli Yaşam Tycoon’un ticari mobil oyun yapısında kullanılabilecek destek para birimidir.



Token oyuncuya başarıyı doğrudan satmayacaktır.



Token’ın amacı:



1\. Bekleme sürelerini azaltmak

2\. Bazı kriz anlarında oyuncuya acil destek vermek

3\. Oyuncunun oyundan kopmasını engellemek

4\. Mobil oyunda kısa oturum temposunu desteklemek

5\. Monetizasyon için adil ve pay-to-win olmayan alan oluşturmak

6\. Oyuncuya “zaman ve konfor satın alma” seçeneği vermek



\---



\## 2. Ana Tasarım Kuralı



Token sistemi şu soruya cevap vermelidir:



> Bu satın alma oyuncuya kolaylık sağlıyor mu, yoksa oyunu onun yerine kazanıyor mu?



Eğer token oyuncuya dünya turunu, 1 milyon takipçiyi veya okyanus hazırlığını doğrudan satın aldırıyorsa sistem başarısızdır.



Ana kural:



```text

Token başarı satmaz.

Token zaman ve kolaylık sağlar.

```

\## 4. Token ile Yapılamayacaklar



Token sistemi pay-to-win olmayacaktır.



Token ile yapılamayacaklar:



1\. Dünya turunu doğrudan tamamlamak

2\. 1 milyon takipçiyi doğrudan satın almak

3\. Okyanus geçişi şartlarını tamamen atlamak

4\. Büyük rotaları risksiz hale getirmek

5\. Tekneyi tek tuşla okyanus hazır yapmak

6\. Sponsor başarılarını kesin garantiye almak

7\. Viral olmayı kesin garantiye almak

8\. Oyuncu skill seviyesini doğrudan maksimum yapmak

9\. Büyük krizleri tamamen yok etmek

10\. Oyunu otomatik kazandırmak



\---



\## 5. Token ve Ana Para Birimi Ayrımı



Oyunda iki ayrı para/değer sistemi olacaktır.



| Sistem | Rol |

|---|---|

| Kredi | Ana oyun içi para birimi |

| Token | Hızlandırma ve kolaylık para birimi |



\## 5.1 Kredi



Kredi, oyuncunun oyun içinde kazandığı ana para birimidir.



Kredi kaynakları:



\- Sosyal medya içerik gelirleri

\- Sponsor gelirleri

\- Görev ödülleri

\- Canlı yayın / bağış sistemi

\- Rota başarı ödülleri

\- İleri sürümde etkinlik gelirleri



Kredi harcama alanları:



\- Tekne satın alma

\- Upgrade alma

\- Bakım yapma

\- Tamir yapma

\- Marina ücreti

\- Yakıt / su / gıda / yedek parça

\- Sigorta veya ekstra hizmetler



\## 5.2 Token



Token, ana başarıyı satmayan destek para birimidir.



Token kaynakları:



\- Oyuncuya başlangıç hediyesi

\- Günlük giriş ödülü

\- Görev ödülü

\- Reklam izleme ödülü

\- İleri sürümde oyun içi satın alma

\- Özel etkinlik ödülü



Token harcama alanları:



\- Süre hızlandırma

\- Acil paketler

\- Kriz etkisini azaltma

\- Günlük fırsat yenileme

\- Bazı konfor kolaylıkları

\## 7. Liman Modunda Token Kullanımı



Liman modu, token kullanımının en güvenli alanıdır.



Çünkü oyuncu burada zaten hazırlık, bakım ve geliştirme yapmaktadır.



| Kullanım | Etki | Sınır |

|---|---|---|

| Küçük upgrade hızlandır | Kurulum süresi azalır | Uygun |

| Büyük upgrade hızlandır | Kurulum süresi azalır | Pahalı olmalı |

| Bakım hızlandır | Bakım süresi azalır | Uygun |

| Tamir hızlandır | Tamir süresi azalır | Uygun |

| İçerik hazırlığını hızlandır | İçerik üretim süresi azalır | Dikkatli kullanılmalı |

| Günlük marina fırsatı yenile | Yeni fırsat gelir | Sınırlı kullanılmalı |



\---



\## 8. Deniz Modunda Token Kullanımı



Deniz modu token kullanımı dikkatli tasarlanmalıdır.



Denizde token oyuncuya kriz anında destek verebilir ama riski tamamen yok etmemelidir.



| Kullanım | Etki | Sınır |

|---|---|---|

| Acil enerji paketi | Enerji krizi geçici çözülür | Tam çözüm değil |

| Acil su paketi | Su riski azalır | Uzun rota için kalıcı çözüm değil |

| Acil yakıt paketi | Motor desteği sağlar | Sınırlı kullanım |

| Kriz müdahale desteği | Hasar etkisi azalır | Hasarı tamamen silemez |

| Seyahat hızlandırma | Süre bir miktar azalır | Okyanusu anında bitiremez |

| Acil iletişim desteği | Yardım / bilgi sağlar | Büyük maliyet veya sınır gerekir |



\---



\## 9. Token ile Seyahat Hızlandırma



Seyahat hızlandırma çok dikkatli olmalıdır.



Kural:



```text

Token seyahati kısaltır ama yolculuk deneyimini yok etmez.

```



Örnek kullanım:



| Rota Tipi | Token Etkisi |

|---|---|

| Kısa rota | 1 oyun günü azaltabilir |

| Orta rota | %10-20 süre azaltabilir |

| Uzun rota | %10-15 süre azaltabilir |

| Okyanus geçişi | Sadece küçük destek verir, geçişi bitirmez |



Okyanus geçişlerinde token:



\- Enerji krizini azaltabilir

\- Su krizini azaltabilir

\- Bazı gecikmeleri düşürebilir

\- Kriz hasarını azaltabilir



Ama:



\- Atlantik geçişini anında bitiremez

\- Pasifik geçişini atlatamaz

\- Hazırlık şartlarını geçersiz yapamaz



\---



\## 10. Token ve Upgrade Hızlandırma



Upgrade hızlandırma token için en sağlıklı kullanım alanıdır.



| Upgrade Tipi | Normal Süre | Token ile Etki |

|---|---:|---|

| Küçük upgrade | 1 oyun günü | Hemen tamamlanabilir |

| Orta upgrade | 2-3 oyun günü | Süre azaltılabilir |

| Büyük upgrade | 4-7 oyun günü | Kısmi hızlandırma |

| Okyanus seviyesi upgrade | 5-10 oyun günü | Sınırlı hızlandırma |



Kural:



> Büyük upgrade’lerde token maliyeti yüksek olmalı ve oyuncuya yine kaynak/para yönetimi yaptırmalıdır.



\---



\## 11. Token ve Bakım / Tamir



Bakım ve tamir, token için doğal kullanım alanıdır.



Oyuncu limana geldiğinde şunu seçebilir:



\- Normal bakım: daha ucuz ama zaman ister

\- Hızlı bakım: token ister, süre azalır

\- Acil tamir: token + kredi ister, hızlı çözüm sağlar



Örnek:



| İşlem | Normal | Token Kullanımı |

|---|---|---|

| Motor bakımı | 2 oyun günü | 1 oyun gününe iner |

| Elektrik tamiri | 3 oyun günü | 1-2 güne iner |

| Yelken onarımı | 2 oyun günü | Hemen veya 1 güne iner |

| Gövde tamiri | 5 oyun günü | 3 güne iner |

\## 18. Token Harcama Öncelikleri



Oyuncuya token kullanımı önerilirken şu öncelik sırası korunmalıdır:



1\. Büyük krizden çıkmak

2\. Oyundan kopmayı engellemek

3\. Sıkıcı bekleme süresini azaltmak

4\. Günlük fırsatları değerlendirmek

5\. Konfor sağlamak



Token asla şu amaçla merkeze alınmamalıdır:



\- Oyuncuyu bilerek sıkmak

\- Oyunu kasıtlı yavaşlatmak

\- Başarıyı paraya bağlamak

\- Oyuncuyu çaresiz hissettirmek

\## 19. Token ve Okyanus Geçişleri



Okyanus geçişleri oyunun en önemli sınavlarıdır.



Token burada dikkatli kullanılmalıdır.



Token yardımcı olabilir:



\- Enerji krizini azaltır

\- Su sorununu geçici çözer

\- Bazı gecikmeleri düşürür

\- Kriz hasarını azaltır



Ama token şunları yapamaz:



\- Atlantik’i anında geçirmez

\- Güvenlik şartını atlatmaz

\- Navigasyon eksiğini yok saymaz

\- Tekneyi hazır hale getirmez

\- Final hedefini satın aldırmaz



Kural:



```text

Okyanus geçişinde token destek olur; kaptanlığın ve hazırlığın yerini almaz.

```

3\. Oyuncuya yanıltıcı paket sunulmayacaktır.

6\. Oyuncu para harcamadan da oyunu tamamlayabilecektir.

7\. Yayın aşamasında mağaza kuralları güncel resmi kaynaklardan tekrar kontrol edilecektir.

\## 23. Token ve Ekonomi Dengesi



Token ekonomisi kredi ekonomisini öldürmemelidir.



Kural:



| Sistem | Ana Rol |

|---|---|

| Kredi | Oyunun ana ilerleme parası |

| Token | Zaman ve kolaylık desteği |



Token çok güçlü olursa kredi sistemi anlamsızlaşır.  

Kredi çok zayıf olursa oyuncu tokena mecbur kalır.



Denge:



```text

Kredi = oyun emeği

Token = zaman kolaylığı

```
