# Mobile UI Skill

## Purpose

Use this guide when changing interface, layout, or presentation in **Yelkenli Yaşam Tycoon**.

## Core UI Direction

- The game is mobile-first.
- UI must be readable on small screens without requiring precise tapping or excessive scrolling.
- Buttons should be large enough for touch interaction.
- Layouts should avoid feeling cramped.
- Important player status should stay visible and easy to understand.

## UI Priorities

- Prefer simple, clean, practical UI improvements.
- Keep the most important actions obvious.
- Reduce clutter before adding new visual elements.
- Preserve clarity of progression, rewards, and current player state.
- Make sure cards, panels, and buttons are comfortably usable on phones.

## Areas To Recheck After UI Changes

- Liman
- Deniz
- Upgrades
- Daily goals
- Arrival reward screen
- Global progress strip

## What To Avoid

- Do not add dense layouts that only work on desktop-sized screens.
- Do not hide critical progression or resource information.
- Do not overdesign simple screens if readability gets worse.
- Do not add flashy UI changes that make the game harder to use.

## Working Style

- Prefer small, testable UI improvements.
- After layout changes, check spacing, text wrapping, button size, and tap comfort.
- Keep visual updates practical and consistent with the game's current structure.
